# FilterNet: Harnessing Frequency Filters for Time Series Forecasting
**NeurIPS 2024** | https://arxiv.org/abs/2411.01623 | Code: aikunyi/FilterNet

---

### 1. Core Idea

FilterNet replaces self-attention with learnable frequency-domain filters that multiply the FFT coefficients of the input directly. Two filter variants are proposed: a Plain Shaping Filter (PaiFilter) with fixed, globally learned weights, and a Contextual Shaping Filter (TexFilter) with input-dependent weights generated by a small network. The claim is that frequency-domain element-wise multiplication subsumes both linear and attention-style mappings while being more robust to high-frequency noise.

---

### 2. Method — Mathematics

**Input normalization:** RevIN on X ∈ ℝ^(N×L).

**Fourier transform:**
```
Z = FFT(X) ∈ ℂ^(N×L)
```

**PaiFilter (plain, universal variant):**
```
S = Z ⊙_L H_φ^(Uni)     H_φ^(Uni) ∈ ℂ^(1×L)   (shared across channels)
```
Individual variant: H_φ^(Ind) ∈ ℂ^(N×L) (per channel).

**TexFilter (contextual, input-dependent):**
```
E = κ(Z)                          # linear map: ℂ^(N×L) → ℂ^(N×D)
H_φ(Z) = σ(E ⊙_D W_{1:K})       # K learnable params W_k ∈ ℂ^(1×D)
S = E ⊙_D H_φ(Z)                 # input-gated filter
```

**Reconstruction and prediction:**
```
S_time = IFFT(S) ∈ ℝ^(N×L)
Y^ = FFN(S_time) ∈ ℝ^(N×τ)      # τ = forecast horizon
```

**Why this ≈ attention:** Circular convolution in time domain = element-wise product in frequency domain. The filter H_φ selectively amplifies/attenuates frequency bands, playing the role of attention's query-key similarity without the quadratic cost.

---

### 3. Architecture / Algorithm

```
X ∈ ℝ^(N×L)
→ RevIN normalization
→ FFT  →  Z ∈ ℂ^(N×L)
→ Filter block (PaiFilter or TexFilter)  →  S ∈ ℂ^(N×L)
→ IFFT  →  S_time ∈ ℝ^(N×L)
→ Feed-Forward Network (2-layer MLP)  →  P ∈ ℝ^(N×τ)
→ RevIN denormalization  →  Y^ ∈ ℝ^(N×τ)
```

For TexFilter the filter block includes the embedding κ and the gating computation before element-wise multiplication.

Stacking: multiple filter+FFN blocks can be chained, but the ablation suggests 1–2 blocks suffice.

---

### 4. What Makes It Work

1. **Full-spectrum utilization:** Standard attention on patches attends to patch-level tokens, effectively blurring fine-grained frequency information. Direct frequency-domain multiplication retains all spectral components.
2. **Noise robustness:** The universal filter H_φ^(Uni) shared across channels acts as a learned global de-noising filter, suppressing high-frequency noise that would otherwise corrupt attention scores.
3. **Channel-shared filter outperforms per-channel:** Ablation (Table 2) shows the universal variant beats the individual variant — sharing forces the filter to learn globally meaningful spectral structure rather than overfitting per-channel noise.

---

### 5. Limitations & Failure Modes

- **High-variate datasets (Traffic, N=862):** iTransformer outperforms FilterNet on Traffic (0.395 vs. 0.506 MSE at horizon 96) — variate-mixing attention is more powerful than pure frequency filtering for high-dimensional channel-dependent data.
- **Circular convolution assumption:** FFT-based convolution is circular; real time series are not. This introduces boundary artifacts at the edges of each window.
- **TexFilter hyperparameter K:** The number of learnable complex vectors K controls expressiveness; too small → underfitting, too large → overfitting. No principled selection rule given.
- **Not tested on:** Irregular sampling, missing values, very short sequences, or probabilistic forecasting.

### 6. Research Ideas (ranked)

**1. Combine FilterNet frequency block with variate attention**
- **Idea:** Use PaiFilter for temporal modeling + variate-axis self-attention (iTransformer style) for cross-channel mixing.
- **Why:** FilterNet fails on high-variate Traffic where channel mixing is critical; a hybrid captures both temporal frequency structure and cross-variate correlations.
- **Test:** Replace temporal attention in iTransformer with PaiFilter; measure Traffic and Weather MSE.

**2. Learnable truncation threshold in PaiFilter**
- **Idea:** Add a learnable soft mask σ(α·|k| + β) to the frequency filter, creating a data-driven low-pass envelope.
- **Why:** The current filter can learn to zero out high frequencies, but not with explicit monotone decay. A learnable envelope biases the filter toward the known fact that low frequencies dominate in time series.
- **Test:** Compare PaiFilter + envelope vs. plain PaiFilter on ETTh1/2; check if the learned envelope resembles a low-pass filter.

**3. TexFilter as attention replacement in Transformer**
- **Idea:** Replace the Q,K,V self-attention layer in a standard Transformer with TexFilter (context-dependent frequency multiplication).
- **Why:** TexFilter is O(N·L) vs. O(N²·L²) for patched attention; the expressiveness of input-dependent gating may compensate for the structural difference.
- **Test:** PatchTST with TexFilter replacing attention; ETTh/Weather benchmark.

**4. Phase-aware filtering**
- **Idea:** Decompose H_φ into separate magnitude and phase components; regularize the phase component toward zero (identity phase).
- **Why:** Arbitrary phase shifts in the learned filter could cause temporal misalignment; constraining phase while freeing magnitude may improve stability.
- **Test:** Ablation: full H_φ vs. magnitude-only H_φ on ETTh1.

**5. Multi-scale FilterNet with different L per scale**
- **Idea:** Apply PaiFilter at multiple temporal resolutions (full L, L/2, L/4) and aggregate predictions.
- **Why:** Different frequency bands are best captured at different resolutions; coarse resolution emphasizes trend, fine resolution emphasizes seasonality.
- **Test:** ETTm datasets (more frequent sampling) where multi-scale structure is richer.

### 7. Implementation Notes

- **FFT convention:** Use `torch.fft.rfft` (real FFT, output length L//2+1) to halve the computation; filter H_φ must have shape (1, L//2+1) accordingly.
- **Complex initialization:** Initialize H_φ to ones (identity filter) for stable training start.
- **TexFilter K:** Paper uses small K (2–4 in practice); larger K risks overfitting on small datasets.
- **Lookback window:** L=96 in primary experiments; longer L=336/512 may improve performance but increases memory.
- **FFN after IFFT:** A 2-layer MLP with GELU activation maps L→τ; this is the main capacity for temporal mixing after filtering.
- **Boundary effects:** Apply a Hann window before FFT to reduce spectral leakage from the rectangular windowing implicit in patching: `X * torch.hann_window(L).unsqueeze(0)`.
- **GPU:** Experiments on RTX 3080; filter operations are fast — the bottleneck is the FFN projection.
